## 修复: SkillHub 安装后不显示在 installed 列表

发现并修复了 2 个 Bug：

**Bug 1** — `clawhub_install` 路由 (`POST /api/clawhub/install`) 安装成功后没有调用 `state.kernel.reload_skills()`
- 文件: `crates/openfang-api/src/routes.rs` 第 3887 行附近
- 安装写入了文件但内存中 SkillRegistry 未刷新，导致 `GET /api/skills` 返回的列表不含新 skill
- 修复: 在 `Ok(result)` 分支开头添加 `state.kernel.reload_skills();`

**Bug 2** — `ClawHubClient::install()` 没有设置 manifest 的 `source` 字段
- 文件: `crates/openfang-skills/src/clawhub.rs` 第 839 行附近
- 转换后的 manifest.source 为 None，前端 `isSkillInstalled()` 通过 `source.type === 'clawhub'` 判断失败
- 修复: 在写入 skill.toml 前设置 `manifest.source = Some(SkillSource::ClawHub { slug, version })`

编译通过，54/55 测试通过（1 个预先存在的失败测试 `test_clawhub_client_old_skillhub_url_is_official` 与此无关）。

## Venus 过载弹性改进

针对 Venus 服务端频繁过载（code 4001 / server_error）导致的空响应和 Provider overloaded 错误，实施了三项改进：

### 改进 1: 增大重试次数和延迟
- `MAX_RETRIES`: 3 → 5（给过载的 provider 更多恢复机会）
- `BASE_RETRY_DELAY_MS`: 1000 → 2000（更长的退避间隔）
- 文件: `crates/openfang-runtime/src/agent_loop.rs`

### 改进 2: 空响应多次重试
- 新增 `MAX_EMPTY_RETRIES = 3` 常量
- 原来空响应（0 in / 0 out）只重试一次，现在最多重试 3 次
- 每次重试带指数退避（2s, 4s, 8s），给 provider 恢复时间
- 同时修改了 non-streaming 和 streaming 两个 agent loop

### 改进 3: classify_error retryable 错误自动重试
- `call_with_retry` 和 `stream_with_retry` 的 `Err(e)` 分支原来直接返回错误
- 现在对 `classified.is_retryable == true` 的错误（如 HTTP 500/503 server_error）自动重试
- 使用 `classified.suggested_delay_ms` 或指数退避作为延迟
- 只有用完所有重试次数或 non-retryable 错误才会返回失败
